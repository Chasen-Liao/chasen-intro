## 2. 分区分类与交互假设

Hero 右侧区域的用途分类为 `identity`，新头像必须使用用户提供的真实素材，不生成替身。

默认交互方案：

1. 首次进入时，粒子从左侧信号流聚合到头像采样位置。
2. 聚合完成后，完整头像提高到可清晰识别的透明度，粒子保留为轻薄印刷/半调质感。
3. 指针靠近时局部粒子产生可逆排斥；青色粒子比普通粒子多一层轻微光晕或短拖尾。
4. 指针离开后粒子回到采样位置，不增加点击爆炸、自动循环解体或全屏粒子。
5. `prefers-reduced-motion` 下跳过 Canvas 动画并直接显示头像。

备选方案：仅将青色线路和耳机标志粒子化，人物主体完全静态。该方案性能更稳，但不作为当前默认目标。

## 6. 复刻简报

- 构建负责方：现有单文件静态站 `index.html`，遵循 `personal-website` 主契约和项目既有实现风格。该技能副本引用的 `frontend-app-builder.md` 在本机缺失，因此第五步使用现有站点结构与浏览器逐区 QA 作为等价兜底。
- 主视觉基准：`docs/references/hero-avatar-particles-reference.png`；只更新 Hero 头像区域，其他分区保持原样。
- 全局外框：保留现有固定导航、左侧进度轨、蓝图画框、状态标签与移动端行为。
- 字体锁定：Playfair Display / Noto Serif SC 用于标题，Inter / Noto Sans SC 用于正文，系统 monospace 用于规格标签。
- 全站配色锁定：纸张 `#F4EFE6`、墨黑 `#1A1714`、橙色 `#F0652E`、线框 `#C9C0B2`、弱文字 `#7D756A`。
- 头像配色锁定：青色 `#87E1F0`、粉色 `#F599AD`、暖棕 `#B58D6F`；不让青粉扩散到全站 UI。
- 动效锁定：从左聚合、稳定后清晰显影、指针局部排斥、青色短拖尾、离屏暂停、页面隐藏暂停、reduced motion 直接显示底图。
- 素材边界：唯一生产位图为 `assets/chasen-avatar-smile.png`；参考图仅作 QA；所有粒子、框线、文字与交互留在代码。
- 允许偏差：实际粒子密度可低于参考图；移动端减少外围粒子；不复刻参考图中生成的文字细节。
- QA 目标：桌面端对照 Hero 参考图检查构图、头像占比、标题层级与色彩；移动端检查完整头像、无溢出和动画降级。
- 风险触发：若白底在 `multiply` 下仍形成明显方块，或 Canvas 采样导致面部不可读，必须先调整呈现/采样策略再继续，而不是用滤镜掩盖。
